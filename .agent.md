# Agent Policy — Repository Level

Branching & commits

- Branch name conventions: `feat/<short>-<slug>`, `fix/<short>-<slug>`, `chore/<desc>`.
- Commit message templates: `feat(scope): short summary`, `fix(scope): short summary`, `chore: short summary`.

Pull requests

- PR checklist: tests pass, summary of changes, verification steps, CI green.
- Agent must open PRs; merging to `main` requires a human reviewer and explicit approval.

Permissions

- Allowed: create branches, run builds/tests, modify code and tests, add docs.
- Forbidden: commit secrets, modify GitHub secrets, merge to protected branches.

CI & gating

- All feature PRs must pass the backend `dotnet` tests and frontend `vitest` before merging.
